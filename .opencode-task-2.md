# 任务：实现通知渠道优先级功能

## 需求

支持配置多个通知渠道的发送优先级，主渠道失败时自动切换到备用渠道，提升通知系统的可靠性。

## 实现要点

1. **数据结构扩展** (`data/notification-channels.json`):
   - 渠道数据新增 `priority` 字段（number，1-10，1 为最高优先级）
   - 支持自动排序，优先级高的渠道先发送

2. **后端函数** (`lib/notification-channels.js`):
   - `getSortedChannels()` - 按优先级排序获取启用的渠道
   - `sendWithFallback(message, options)` - 发送消息，支持失败时自动切换到下一个优先级的渠道
   - 记录每个渠道的发送尝试和结果

3. **后端 API** (`server.js`):
   - `PATCH /api/admin/notification-channels/:id/priority` - 更新渠道优先级
   - `POST /api/admin/notification-channels/reorder` - 批量重排优先级（拖拽排序）
   - 扩展 `PUT /api/admin/notification-channels/:id` 支持 priority 字段

4. **前端 UI** (`public/app.js`, `public/index.html`, `public/styles.css`):
   - 渠道列表显示优先级徽章
   - 支持拖拽排序或上移/下移按钮调整优先级
   - 显示"主渠道"、"备用渠道"标签
   - 发送测试消息时显示优先级顺序
   - 深色主题适配，响应式布局

5. **通知发送链路改造**:
   - 所有通知发送函数（备份通知、负载预警、任务完成通知等）改为使用 `sendWithFallback`
   - 记录发送历史，包括尝试的渠道顺序和最终成功的渠道

6. **审计事件** (`lib/audit.js`):
   - `notification_channel.priority_changed` - 优先级变更
   - `notification_channel.sent_with_fallback` - 使用备用渠道成功发送
